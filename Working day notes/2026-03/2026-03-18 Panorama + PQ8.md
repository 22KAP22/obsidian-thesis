I finally managed to make Panorama + PQ8 work. Panorama with a flat ivf was tested using the official distribution of FAISS. The results seem promising. The Pano x PQ implementation I used was the one from the author's of the Panorama paper. It's quite intuitive to use. The only problem was that they included deep in their code some intrinsics which were only available for sapphire rapids and newer intel processors. With the help of chatGPT I managed to rewrite the code in a generic way. I don't know how much performance get's lost here. Precision and memory footprint should be the same or very close to the original. Maybe I should recompile it with -march and -mtune 'cascadelake'.

Big problem I have is that I can't store the indexes. So for each run the index needs to be rebuilt. This takes quite some time. The problem lies with the serialization of the new data structures and variables added to the new panorama index or rather the lack thereof.
Currently building and then immediately using the index. I don't know what happens to the index once it's not used anymore. I hope the python garbage collector recognizes it's not used anymore and frees up the space, since these are rather big.
